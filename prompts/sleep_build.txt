# Rules & Constraints
请在提取关系时严格遵守以下准则：
1. 【实体边界限制】：绝对禁止制造、捏造或引入不在上述列表中的新实体。所有输出关系的 `source` 和 `target` 字段值必须与列表中给出的名称完全一致，字面不可做任何增删改。
2. 【内容安全限制】：为了防止 JSON 字符串解析冲突，`source`、`target`、`relation_type` 和 `description` 中的文本内容【绝对不能包含任何单双引号或反斜杠】。
3. 【避免无效关联】：禁止输出自环关系（即 `source` 和 `target` 是同一个实体）。如果两个实体之间有多重关系，请提炼成最核心的一条，不要输出重复的实体对。
4. 【语言规范】：所有字段的文本描述（如 `description`、`time` 等）必须统一使用【简体中文】。

# Relation Types List
请根据语义，必须从以下限定的关系清单中选择最精准的一个填入 `relation_type` 字段（不得自行发明新类型）：
- belongs_to: 人物、官员等隶属于某个组织、国家或团队
- subsidiary_of: 某个组织、机构、部门等隶属于上级组织、集团或国家
- invest_in: 投资、金融注资、控股关系
- partner_with: 伙伴、结盟、签署协议或合作关系
- compete_with: 竞争、对抗、敌对、制裁或冲突关系
- leads_to: 导致、引发或触发了某事件/连带风险
- deployed_to: 部署、驻扎、调动到某地（如军队部署、武器配置、资金转移到某国）
- developed_by: 某技术、武器、资产由某组织/人研发
- equipped_with: 某部队或国家装备/配置了某武器或设备
- participate_in: 参与、出席或加入了某行动、演习、会议或事件
- monitored_by: 被某组织、国家、卫星或系统监视/监控/调查
- founded_by: 某组织、机构或事件由某人/组织创立
- related_to: 只有在上述关系完全无法精确匹配，但两者确实存在某种强关联时作为兜底选项
